How Servers Work

Servers allow us to create, share, and display web content. Using a public web sever allows anyone with access to the internet to view a specified web site and contact. Servers render HTTP in a way that browsers can display the content correctly to the user

The purpose of a server is to respond to HTTP requests. Servers can display static (pre-written content files) and Dynamic (content requests that are build real time).

Web applications are programs that build web content.

This video does a good job of explaining more about servers.

Importance of Validating Input

Validating input allows the server side that the input received is what we expected. Without validation, users could unintentionally or intentionally cause malicious changes to our site.

This video does a good job of explaining more about validation

HTML Templates and Abstraction

A template library is a library to build complicated strings. When developing web applications the use of templates allows us to breakup and simplify our code within many abstracted code documents.

Templates also help in avoiding repetition by allowing us to call on and reuse sets of code without having to retype.

In our shopping list exercise we used a common base or index html page to display the header and footer. Additional pages of code where called using extends and block content. This allowed us to easily build a dynamic web page without having to use redundant code.
